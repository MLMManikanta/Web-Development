CSS Layout

CSS Layout controls how elements are arranged on a webpage ‚Äî including their display behavior, alignment, and positioning. The foundation of any web design starts with understanding the display property.

1. The display Property

The display property defines how an element behaves in the document flow. It determines whether an element acts as a block, inline, or inline-block element ‚Äî or if it‚Äôs hidden completely.

4. Float and Clear

The float property positions elements to the left or right of their container, allowing inline content (like text or other boxes) to wrap around them. Originally used for text wrapping around images, it's now less common due to Flexbox and Grid.

4. Float and Clear

The float property lets elements "float" to the left or right side of their container, allowing other content like text or inline elements to wrap around them. The clear property then controls how other elements behave around floated items.

4.1 The float Property

The float property removes elements from the normal document flow and positions them to the left or right of their container. Commonly used for images or side boxes.

  • float: left ‚Äì aligns element to the left and allows text to wrap on the right.
  • float: right ‚Äì aligns element to the right and allows text to wrap on the left.
  • float: none ‚Äì disables floating (default behavior).

Example 1 ‚Äì float: left

This image is floated to the left of the text. The paragraph wraps naturally around it, thanks to float: left;. 


img {
  float: left;
  margin-right: 12px;
}

Example 2 ‚Äì float: right

Here, the image is floated to the right. The text wraps on the opposite side, maintaining a natural flow. 


img {
  float: right;
  margin-left: 12px;
}

Example 3 ‚Äì float: none

This image uses float: none;, meaning it stays in the normal document flow without text wrapping around it. 


img {
  float: none;
  display: block;
  margin: 0 auto;
}

4.2 The clear Property

The clear property prevents elements from wrapping around floated siblings. It tells the browser ‚Äústart below floated elements‚Äù on the specified side.

  • clear: left ‚Äì moves element below any left-floated items.
  • clear: right ‚Äì moves element below any right-floated items.
  • clear: both ‚Äì moves element below all floated items (most common).

Example 1 ‚Äì clear: left

  clear: left;
}

Example 2 ‚Äì clear: right

Float Right

This paragraph clears the right float.

  clear: right;
}

Example 3 ‚Äì clear: both

Float Left
Float Right

This text clears both floated boxes.

  clear: both;

4.3 Clearfix (Automatic Float Containment)

When child elements are floated, their parent might collapse because floats are taken out of normal document flow. The clearfix method forces the parent to wrap around floated elements.

6. The gap Property

The gap property controls the spacing between items inside layout containers like flex, grid, or multi-column elements. It eliminates the need for adding individual margins and ensures clean, consistent spacing.

There are three main properties:

  • gap ‚Äî shorthand for both row and column gaps.
  • row-gap ‚Äî defines vertical spacing between rows.
  • column-gap ‚Äî defines horizontal spacing between columns.

The gap property defines uniform spacing between both rows and columns. It can be used with flex, grid, or multi-column layouts.

7. CSS Positioning

CSS positioning defines how elements are placed within the web page layout. Using it properly allows you to move, fix, or overlap elements as needed. Let‚Äôs explore each type of positioning with examples and visual containers.

Static is the default positioning for all elements. Elements are displayed in the order they appear in the HTML ‚Äî no top, left, or other offsets are applied.

7.2 Relative Positioning

The relative value positions elements relative to their normal position. The element still occupies its original space, but you can move it slightly using top, left, right, or bottom.

Normal Box
Moved (top: 25px; left: 40px)
Another Box

  position: relative;

  top: 25px;
  left: 40px;

Notice that even though the second box is moved, the others don‚Äôt adjust their positions ‚Äî it still keeps its original space.

7.3 Absolute Positioning

Absolute positioning removes the element from normal flow and places it relative to the nearest ancestor that has position: relative;.

Parent Container (position: relative)

Absolutely Positioned

.absolute-parent {
  position: relative;
}
.abs-child {
  position: absolute;
  top: 30px;
  right: 30px;
}

7.3.1 Use Cases

  • Placing icons, badges, or overlays.
  • Creating popups, modals, or dropdown menus.

7.3.2 Pitfalls

  • Can escape container bounds if the parent is not positioned.
  • Overusing may break responsive layouts.

8. CSS Media Queries

Media Queries allow your webpage to adapt its layout and styling depending on the device‚Äôs screen size, orientation, or resolution. This is the foundation of responsive web design ‚Äî making pages look good on mobiles, tablets, and desktops.

8.1 Basic Syntax

The @media rule defines different styles for different screen sizes or device types. 


@media (condition) {
  /* CSS styles go here */
}

Example: Applying styles for screens smaller than 768px (like mobile devices):


@media (max-width: 768px) {
  body {
    background-color: #f0f9ff;
  }
}

8.4 Orientation Example

You can use orientation in media queries to apply styles based on whether the device is in portrait or landscape mode.

Rotate your screen to see changes in background color.


@media (orientation: landscape) {
  .orientation-demo {
    background-color: #007acc;
    color: white;
  }
}

8.5 Combining Conditions

You can combine multiple conditions using and, or, and not operators. 


@media (min-width: 768px) and (max-width: 1024px) {
  .container {
    background: lightblue;
  }
}

@media not all and (orientation: landscape) {
  body {
    background: #f5f5f5;
  }
}

The first example targets tablets (between 768px and 1024px). The second excludes devices in landscape orientation.
