GitHub – Complete Guide

GitHub is a web-based platform for version control and collaboration. It allows developers to store, track, and manage code using Git.

What is GitHub?

  • GitHub is a hosting platform built on Git.
  • It allows multiple developers to work on the same project efficiently.
  • GitHub provides tools for code review, collaboration, and issue tracking.
  • You can use it for open-source or private repositories.

1. Creating a GitHub Account

  • Visit https://github.com
  • Sign up with your email and password.
  • Verify your email address.

Create a New Repository

  • Click the “+” icon → New repository
  • Enter repository name (e.g., my-first-repo)
  • Add an optional description.
  • Choose Public or Private.
  • Check “Initialize this repository with a README” (optional).
  • Click Create repository.

Rendered Output Example:

Repository: my-first-repo
Description: Learning GitHub Basics
Visibility: Public

2. Connecting Git with GitHub

Step 1: Configure Git (one time only)

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Step 2: Create a Local Folder

mkdir my-first-repo
cd my-first-repo
git init

Step 3: Link Local Repo to GitHub

git remote add origin https://github.com/username/my-first-repo.git

Step 4: Add and Commit Files

echo "Hello GitHub!" > README.md
git add README.md
git commit -m "Initial commit"

Step 5: Push to GitHub

git push -u origin main

3. Cloning a Repository

Cloning means creating a local copy of a GitHub repository on your computer. 

git clone https://github.com/username/project-name.git
cd project-name

Rendered Output Example:

Cloning into 'project-name'...
remote: Enumerating objects: 10, done.
Receiving objects: 100% (10/10), done.

4. Branching in GitHub

  • Branch: A version of your project where you can make changes safely.
  • main is the default branch.
  • Use branches to develop new features without breaking the main code.
git branch feature-login
git switch feature-login
# make changes
git add .
git commit -m "Add login feature"
git push -u origin feature-login

Rendered Output Example:

Branch created: feature-login
Switched to branch 'feature-login'

5. Pull Requests (PR)

  • Pull requests let you propose changes from one branch to another.
  • They enable team review, discussion, and approval before merging.

Steps:

  1. Push your branch to GitHub.
  2. Go to the repository → “Pull requests” → “New pull request”.
  3. Select the branch to merge into main.
  4. Review and click Create pull request.

Example Description:

Title: Added login feature
Description: Implemented user login with validation
Reviewer: @project-lead

6. Issues and Discussions

  • Issues: Used to track bugs, tasks, or feature requests.
  • Discussions: Used for brainstorming or asking questions.

Example Issue:

Title: Bug in login validation
Description: Empty username field not showing error message.
Labels: bug, frontend

7. Forking a Repository

  • Fork: Creates your own copy of someone else’s repository.
  • Useful for contributing to open-source projects.

Example:

# Fork a repo via GitHub UI, then clone your fork:
git clone https://github.com/yourusername/forked-repo.git

8. Contributing to a Repository

  • Fork the repository.
  • Clone it to your computer.
  • Create a new branch for your feature.
  • Make changes and push them.
  • Submit a Pull Request to the original repository.

Example Command Flow:

git clone https://github.com/yourusername/project.git
cd project
git switch -c feature-update
# make changes
git add .
git commit -m "Improved documentation"
git push -u origin feature-update

9. README and LICENSE

  • README.md — Explains project purpose, setup steps, and usage.
  • LICENSE — States how others can use your code (e.g., MIT License).

Example README.md content:

# My Project
A simple GitHub demo repository.

## Features
- Git basics
- Branch management
- Collaboration workflow

## License
MIT License

10. Summary

  • GitHub = Collaboration + Git + Cloud hosting.
  • Create → Clone → Commit → Push → Pull Request → Merge.
  • Use Issues and Discussions for communication.
  • Keep documentation updated (README, LICENSE).
