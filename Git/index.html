1) Install & First-Time Setup

git --version

Verifies that Git is installed and prints the installed version.

git --version
git config --global user.name / user.email

Sets your identity for all repositories on this machine (stored in ~/.gitconfig).

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main

Controls which branch name new repos start with. Many teams use main.

git config --global init.defaultBranch main
git config --list --global

Shows all globally configured Git settings so you can confirm your setup.

git config --list --global

2) Core Concepts

  • Working Tree: your files on disk right now.
  • Index/Staging Area: what will go into the next commit.
  • Commit: a snapshot of staged changes.
  • Branch: a movable pointer to a commit.
  • HEAD: your current branch/commit.
  • Remote: a server copy (e.g., GitHub) referenced by a name like origin.
  • Tag: a named pointer (used for releases).

3) Start or Clone a Repository

git init — create a new repository

Initializes a new Git repository in the current directory by creating a .git folder.

mkdir myproject && cd myproject
git init
git clone <url> — copy an existing repository

Downloads a repo and its history to a new folder.

git clone https://example.com/user/repo.git
cd repo

4) Daily Workflow

git status

Shows staged, unstaged, and untracked changes to help you decide what to add/commit.

git status
git add

Moves changes into the staging area. Use paths for precision or . to add everything.

git add path/to/file
git add .
git commit -m "message" / git commit --amend

commit saves the staged snapshot. --amend edits the last commit (message or content).

git commit -m "feat: explain daily workflow"
git commit --amend

5) Inspecting Changes & History

git diff / git diff --staged

Shows line-by-line differences in your working tree vs. index, or staged vs. last commit.

git diff
git diff --staged
git log (common views)

Browse commit history. Options can simplify or visualize the graph.

git log --oneline --graph --decorate --all
git show HEAD         # show the last commit details
git blame <file> / git log --follow -- <file>

blame shows who last changed each line; --follow tracks a file across renames.

git blame src/app.js
git log --follow -- src/app.js

6) Branching

git branch / git switch

Create, list, or move between branches. switch -c creates and checks out in one step.

git branch
git switch -c feature/login
git switch main

7) Merge & Rebase

git merge <branch>

Combines another branch into the current branch by creating a merge commit (unless fast-forward).

git switch main
git merge feature/login
git rebase <base>

Replays your current branch’s commits on top of a new base for linear history. Avoid rebasing shared history.

git switch feature/login
git rebase origin/main
# fix conflicts → git add <files> → git rebase --continue

8) Working with Remotes

git remote -v / git remote add origin <url>

Shows configured remotes or adds a new one named origin.

git remote -v
git remote add origin https://example.com/user/repo.git
git fetch / git pull / git push

fetch updates remote tracking branches. pull = fetch + merge (or rebase). push uploads commits.

git fetch origin
git pull
git push -u origin main
Upstream tracking

Connect a local branch with a remote one so git push and git pull know where to go.

git branch --set-upstream-to=origin/main main

9) Tags & Releases

git tag (lightweight vs annotated)

Tags mark specific commits—often for releases. Annotated tags store a message and metadata.

git tag v1.0.0
git tag -a v1.0.0 -m "First release"
git push --tags

10) Ignoring Files

.gitignore

Prevents matching files from being tracked (but doesn’t remove already-tracked files).

# .gitignore examples
node_modules/
dist/
.env
.DS_Store
*.log
Stop tracking files now ignored

Remove from index, keep in working tree, and commit the change.

git rm -r --cached .
git add .
git commit -m "Stop tracking ignored files"

11) Stashing

git stash

Temporarily shelves local changes without committing so you can switch branches or pull cleanly.

git stash push -m "WIP: login form"
git stash list
git stash show -p stash@{0}
git stash apply stash@{0}
git stash pop

12) Undo & Repair

git restore (working tree / index)

Discard local changes to files or unstage changes you added by mistake.

git restore path/to/file          # discard working tree changes
git restore --staged path/to/file # unstage
git revert <commit>

Safely undoes a commit by creating a new commit that reverses it—good for shared branches.

git revert 1a2b3c4d
git reset (soft/mixed/hard)

Moves the current branch pointer. Soft keeps changes staged; mixed keeps them unstaged; hard discards them (dangerous).

git reset --soft HEAD~1
git reset HEAD~1          # mixed (default)
git reset --hard HEAD~1   # destructive
git reflog

Shows where HEAD has been. Use it to find “lost” commits after a reset or rebase.

git reflog

13) Cherry-Pick & Bisect

git cherry-pick <commit>

Applies specific commits from elsewhere onto your current branch—ideal for hotfixes.

git switch hotfix
git cherry-pick 1a2b3c4d
git bisect

Binary searches history to find the first bad commit that introduced a bug.

git bisect start
git bisect bad
git bisect good v1.0.0
# Run tests, mark as good/bad repeatedly, then:
git bisect reset

14) Worktrees & Sparse Checkout

git worktree

Work on multiple branches simultaneously in separate folders without re-cloning.

git worktree add ../repo-feature feature/login
git worktree list
git worktree remove ../repo-feature
git sparse-checkout

Checkout only parts of very large repositories to save time and disk space.

git clone --no-checkout https://example.com/huge.git
cd huge
git sparse-checkout init --cone
git sparse-checkout set path/to/subdir
git checkout main

15) Submodules

git submodule

Embed another Git repo as a dependency at a specific commit; great for vendored libraries.

git submodule add https://example.com/lib/engine.git external/engine
git submodule update --init --recursive

16) Git LFS (Large Files)

git lfs

Stores large or frequently changing binary files outside normal Git objects to keep repos small.

git lfs install
git lfs track "*.psd"
git add .gitattributes
git commit -m "Track PSD files with LFS"

17) Search & Useful Logs

git grep "pattern"

Searches tracked files’ content across the repo—fast and version-aware.

git grep "TODO:"
Diffs between refs

Compare commits, branches, or tags to see what changed.

git diff v1.0.0..v1.1.0

18) Config & Aliases

Helpful aliases

Shortcuts to speed up common tasks. They live in your global config.

git config --global alias.st "status -sb"
git config --global alias.lg "log --oneline --graph --decorate --all"
git config --global alias.last "show -p --stat HEAD"
Credential helpers

Save or manage credentials for seamless pushes/pulls.

# macOS:
git config --global credential.helper osxkeychain
# Windows:
git config --global credential.helper manager
# Linux (1 hour cache):
git config --global credential.helper "cache --timeout=3600"

19) Troubleshooting Tips

  • Detached HEAD: create a branch to keep work — git switch -c rescue-work
  • Non-fast-forward push: pull (merge/rebase), resolve, then push.
  • Secrets committed: rotate keys, remove from history (e.g., git filter-repo), and force-push carefully.
  • Large binaries: use Git LFS.
